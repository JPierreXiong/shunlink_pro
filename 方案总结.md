# 📋 SoloBoard 优化方案总结

## 🎯 两大核心方案

### 方案 1: QStash 调度优化

#### 调度策略
```
免费用户: 每天 2 次（9:00, 21:00 UTC）
付费用户: 每天 8 次（每 3 小时）
```

#### 成本控制
```
100 用户: $4.80/月
500 用户: $24/月
1000 用户: $48/月
```

#### 技术要点
- 支持计划过滤 (`?plan=free` 或 `?plan=paid`)
- 按需同步（只同步有 API 配置的站点）
- 频率控制（免费 12 小时，付费 3 小时）

### 方案 2: 支付系统区分

#### Creem 双重角色
```
角色 1: SoloBoard 收款
- 用途: 用户购买订阅
- API Key: SoloBoard 的 Creem 账号
- 存储: subscription 表

角色 2: 用户销售额监控
- 用途: 监控用户的 Creem 店铺
- API Key: 用户提供的 Creem API Key
- 存储: monitored_sites.apiConfig
```

#### 其他支付方式
```
Stripe/PayPal: 
- 可用于 SoloBoard 收款
- 可用于监控用户销售额

Lemon Squeezy/Shopify:
- 仅用于监控用户销售额
```

## 📊 数据流向

### SoloBoard 收款流程
```
用户选择计划 → 选择支付方式 (Creem/Stripe/PayPal)
    ↓
使用 SoloBoard 的支付账号
    ↓
收款到 SoloBoard
    ↓
记录到 subscription 表
    ↓
解锁功能（免费 → 付费）
```

### 用户销售额监控流程
```
用户配置销售平台 API Key
    ↓
保存到 monitored_sites.apiConfig
    ↓
QStash 定时触发同步
    ↓
使用用户的 API Key 获取数据
    ↓
显示在 Dashboard
```

### 2. API 端点优化

```typescript
// src/app/api/cron/sync-sites/route.ts

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url);
  const planType = searchParams.get('plan'); // 'free' | 'paid'
  
  // 查询对应计划的站点
  const sites = await db()
    .select()
    .from(monitoredSites)
    .innerJoin(user, eq(monitoredSites.userId, user.id))
    .where(
      planType === 'free' 
        ? eq(user.planType, 'free')
        : inArray(user.planType, ['base', 'pro'])
    );
  
  // 过滤：只同步有 API 配置的站点
  const sitesToSync = sites.filter(site => {
    const apiConfig = site.apiConfig as any;
    return apiConfig && Object.keys(apiConfig).length > 0;
  });
  
  // 执行同步...
}
```

### 1. API Key 隔离
```typescript

const soloboardKey = process.env.CREEM_API_KEY;  // SoloBoard 收款
```
